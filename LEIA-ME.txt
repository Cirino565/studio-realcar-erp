STUDIO REALÇAR — VERSÃO CONSOLIDADA

ANTES DE COMEÇAR

1. Faça uma cópia da pasta estável atual.
2. Extraia este ZIP sobre uma cópia do projeto, confirmando a substituição dos arquivos.
3. Confirme que o terminal está aberto na pasta correta:

   git rev-parse --show-toplevel

4. Não atualize Prisma, npm ou outras dependências nesta etapa.
5. Não use prisma db push. Esta entrega não exige alteração de schema ou banco.

ARQUIVOS DE AMBIENTE

Por segurança, os arquivos .env e .env.local com credenciais não são incluídos no ZIP final. Ao extrair sobre o projeto existente, mantenha os seus arquivos atuais e as variáveis configuradas no Railway. Use apenas os arquivos *.example como referência.

VALIDAÇÃO OBRIGATÓRIA

Execute:

npm run build

Avisos já conhecidos e não bloqueadores:

- package.json#prisma será removido apenas no Prisma 7.
- A convenção middleware será substituída futuramente por proxy.
- O npm pode informar que existe uma versão mais recente.

Não corrija esses avisos agora. O importante é o build terminar com sucesso.

PUBLICAÇÃO MANUAL

Após o build passar:

git add .
git commit -m "finalize premium crm experience"
git push

PUBLICAÇÃO ASSISTIDA NO WINDOWS

Também é possível executar:

PowerShell -ExecutionPolicy Bypass -File .\FINALIZAR_E_PUBLICAR.ps1

O script executa o build, cria o commit quando houver alterações e envia a branch atual ao GitHub.

RAILWAY

1. Aguarde o deploy automático do commit novo.
2. Confira a mensagem/hash do commit mostrado pelo Railway.
3. Se necessário, use “Deploy Latest Commit”.
4. Evite “Redeploy” sobre um deploy antigo, pois ele republica a mesma versão antiga.
5. Teste em aba anônima ou atualize sem cache quando o navegador mantiver a interface anterior.
